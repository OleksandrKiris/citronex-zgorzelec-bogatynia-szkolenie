# Logika lokalizacji - Zgorzelec / Bogatynia

Ten projekt laczy dwie lokalizacje, ale ich logika pracy nie jest taka sama.

## Zgorzelec

- Dotyczy tylko magazynu bananowego.
- Magazyn bananowy ma osobny system pracy.
- Nie nalezy pokazywac Zgorzelca jako szklarni.
- Nie nalezy kierowac pracownika Zgorzelca do instrukcji reader/tablet z Bogatyni, jezeli przelozony nie potwierdzi inaczej.

## Bogatynia

- Dotyczy szklarni i magazynu.
- Obowiazuje system Priva.
- W Bogatyni trzeba pokazywac instrukcje readera i tabletu.
- Modul szklarni, readera i tabletu dotyczy Bogatyni.

## Zasada UX

Pracownik ma od razu zrozumiec:

1. Zgorzelec = magazyn bananowy, osobny system.
2. Bogatynia = szklarnia + magazyn, reader + tablet.

Nie mieszamy tych instrukcji na jednym ekranie bez jasnego oznaczenia lokalizacji.
